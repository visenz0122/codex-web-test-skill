# 异常路径(通用)

## 适用范围

**几乎所有非纯静态的功能都应该匹配此模式**——异常路径是通用的,不专属于某种业务,
但是最常被遗漏的一类测试。

不适用的场景很少,只有:

- 纯静态展示页(无服务器交互)
- 纯前端组件(tab 切换、tooltip)

## 必查清单

### 网络异常

- **请求时网络断开**:UI 是否有提示、是否能重试
- **慢网络**(每段延迟很大):loading 状态、超时设置
- **网络抖动**:连接断了又恢复,功能能否继续
- **完全离线**:offline 模式下的体验
- **代理 / VPN 引发的奇怪行为**:某些代理会改 HTTP 头、缓冲响应

### 服务器异常

- **HTTP 5xx**:500 / 502 / 503 / 504,前端是否优雅降级
- **HTTP 4xx**:400(请求格式错)、401(未认证)、403(无权限)、404、409(冲突)、429(限流)
- **非预期响应格式**:JSON 解析失败、空响应、HTML 而非 JSON
- **响应字段缺失**:期望字段不存在
- **响应字段类型错**:期望 number 实际 string

### 资源耗尽

- **限流触发**(429):用户 / IP / 接口 / 全局限流
- **配额超限**:用户已达上限(存储、API 调用、记录数)
- **磁盘满 / 内存不足**:服务器侧资源耗尽
- **数据库连接池满**:并发请求超过池大小

### 数据异常

- **并发修改冲突**(409):乐观锁失败、版本不匹配
- **唯一性约束违反**:重复插入
- **外键约束违反**:引用不存在的资源
- **数据类型不匹配**

### 第三方依赖失败

- **邮件服务挂了**:发送验证邮件失败如何处理
- **支付网关超时 / 失败**:订单状态如何处理(挂起?回滚?)
- **外部 API 失败**:如调用地图 API、调用 LLM API、调用 OAuth 提供商
- **CDN 失败**:静态资源加载失败,UI 能否优雅降级
- **第三方服务慢**:超时、重试、降级策略

### 超时

- **请求超时**:前端 fetch 超时、后端调下游超时
- **长任务超时**:定时任务、异步任务的超时
- **会话中途超时**:用户操作中途 session 过期

### 浏览器 / 客户端环境

- **浏览器 tab 被冻结**(后台 tab)
- **本地存储满**(localStorage / cookie 配额)
- **不同浏览器差异**(若需跨浏览器支持)

### 认证失效中途

- **session 中途过期**:用户操作中途 session 过期,体验如何
- **token 刷新失败**:refresh token 也过期
- **用户被踢下线后操作**:如何提示

## 关键提示

**"异常路径完全没测"是最常见的 P0 漏测**——开发者倾向于"happy path 偏好",
异常路径如果不强制提醒,几乎一定会被忽略。

**所有功能(除纯静态)都应该至少匹配这个模式的几条**——
不需要测全部,但**至少**要测:
- 网络断开
- 后端 5xx
- 输入校验失败的服务器响应

**模拟异常的工具支持**:
- Playwright 支持 offline / route mock / 状态码注入,完整支持各种异常模拟
- Browser Use 支持有限——通常只能等真实异常发生
- 如果工具不支持,应在自检表标 ⚠ + 工具能力理由

## 常见叠加

这个模式**几乎和所有其他模式叠加**——任何主路径模式都应该配套异常路径模式。

特别注意:
- **登录功能**:密码错、限流、CAPTCHA 失败、账号锁
- **支付功能**:支付网关超时、回调失败、并发支付
- **文件上传**:文件过大、格式错、断网
- **LLM agent**:LLM 服务挂、超时、限流

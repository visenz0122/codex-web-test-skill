# 用户认证 / 会话管理

## 适用范围

任何涉及"用户身份建立、维持、终止"的功能:

- 登录(密码 / SSO / 魔法链接 / 二步验证)
- 登出(单端 / 全端)
- 记住登录(持久 cookie / refresh token)
- 跨标签同步 / 跨设备同步
- 会话过期、token 刷新
- 账号锁定、解锁

判别要点:**功能的核心是改变或维持"用户的登录态"**。

## 必查清单

### 登录正路径

- 已注册用户 + 正确密码 → 登录成功
- SSO 登录(如有 Google / GitHub)
- 魔法链接登录(如有)
- 二步验证流程(如有)

### 登录失败路径

每种失败原因独立测:

- 密码错误
- 账号不存在
- 账号已锁定
- 限流触发
- CAPTCHA 失败(如有)
- 网络异常
- 后端 500

### 会话生命周期

- 刚登录:cookie 设置正确、HttpOnly + Secure + SameSite
- 即将过期:刷新机制是否触发
- 已过期:用户操作时如何处理
- 被强制下线:管理员踢人、密码修改后所有 session 失效

### 跨标签 / 跨设备同步

- A 标签登出,B 标签是否同步登出
- 多设备同时登录:是否互踢(取决于业务设计)
- A 设备改密码,B 设备的 session 是否失效

### 安全 invariant

- **token 标志**:HttpOnly、Secure、SameSite 都设了吗
- **登录态泄露**:URL 含 token 吗、localStorage 含密码吗、日志含密码吗
- **会话固定攻击**:登录前后 session id 是否变化
- **暴力破解防护**:同 IP / 同账号失败次数限制
- **密码安全**:存储是否哈希、传输是否 HTTPS
- **邮箱枚举防护**:密码错和账号不存在的响应是否一致(状态码、消息、响应时间)

### 登出

- 主动登出:cookie 清除、服务器侧 session 撤销
- 多端登出:同账号其他 session 是否失效
- 登出后访问需登录页:正确跳转登录页

## 关键提示

**邮箱枚举防护**是最常被忽略的安全测试点之一。开发者直觉上写"密码错"和"账号不存在"
两种不同提示——这恰恰泄露了邮箱是否注册。Inspector 看到 B2 / B3 响应不一致要敏感。

**会话同步问题**在测试时容易被忽略——通常需要两个浏览器或两个标签同时操作才能发现,
人工测试很容易跳过这一步。

## 不适用情况

- 单纯展示登录态的页面(只读用户信息)→ 用"个人主页 / 资料管理"模式
- 权限检查(已登录后能不能做某事)→ 用"多租户 / 权限矩阵"模式

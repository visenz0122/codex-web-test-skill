# CRUD 列表与详情

## 适用范围

资源管理类:

- 文章 / 博客 / 帖子
- 订单 / 商品 / 库存
- 任务 / 项目 / 工单
- 评论 / 回复
- 日程 / 事件 / 提醒
- 文件 / 文档 / 文件夹

判别要点:**功能围绕一类"资源"做增删改查**。

## 必查清单

按 C / R / U / D 分四块,加列表(L)。

### List(列表)

- **分页边界**:第一页、最后一页、超出末页(N+1)、空集
- **筛选 / 搜索**:每个筛选字段单独 + 组合 + 空查询、特殊字符查询
- **排序**:每个可排序字段升序 + 降序 + 默认排序
- **大数据集**(若业务有性能要求):查询响应时间断言
- **越权列表**:能否看到不该看的资源(如别的租户、别的部门)

### Read(详情)

- **存在的资源**:正常读取
- **不存在的资源**:返回 404 而非 500
- **已删除的资源**(软删除场景):返回 410 / 404 / 显示"已删除"
- **越权读取**:别人的资源、不同租户的资源 → 应返回 403 或 404

### Create(创建)

- **正常创建**:数据完整
- **必填字段缺失**:每个必填字段单独缺
- **唯一性约束冲突**:已存在同名 / 同 ID
- **关联资源不存在**:外键引用了不存在的资源
- **批量创建**(若有):部分成功部分失败的事务行为
- **超出配额**(若有):用户已达资源数量上限

### Update(更新)

- **全量更新 vs 部分更新**(PUT vs PATCH)
- **并发修改冲突**:乐观锁、悲观锁、最后写入获胜的策略
- **更新已删除资源**:应被拒绝
- **越权更新**:更新别人的资源 → 应被拒绝
- **不可变字段**:某些字段创建后不允许修改(如订单的 created_at)
- **状态相关的更新限制**:已支付订单的某些字段不能改

### Delete(删除)

- **正常删除**
- **删除不存在的资源**:幂等性(404 还是 200,要看 API 设计)
- **删除已删除的资源**(软删除场景)
- **级联删除**:删父资源时子资源如何处理(级联删 / 标记孤儿 / 拒绝删)
- **软删除恢复**(若有):删除后能否恢复、恢复后状态正确吗
- **越权删除**:别人的资源 → 应被拒绝
- **批量删除**:部分成功部分失败的事务行为

## 关键提示

**越权操作**是 CRUD 最容易遗漏的——开发者只测"自己 CRUD 自己的资源"。
权限测试必须**穷举**:[自己 / 别人] × [C / R / U / D] = 8 个组合,每个都要测被拒。

**软删除**是个常见陷阱——前端"看不见"删除的资源,但 API 直调能恢复 / 绕过。
测试时要从 API 层验证,不只看 UI。

**幂等性**——删除不存在的资源,API 设计上有两种合理选择(404 / 200),
但**必须在规约里明确**,不能让 Cartographer 随便测。

## 常见叠加

通常和这些模式一起出现:
- 多租户 / 权限矩阵(几乎一定)
- 状态流转(订单等有状态的资源)
- 异常路径(通用)
- 表单输入型(创建 / 更新本身是表单)

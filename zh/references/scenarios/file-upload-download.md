# 文件上传 / 下载

## 适用范围

任何文件输入或文件输出的功能:

- 头像 / 图片上传
- 文档上传(PDF / Word / Excel)
- 视频 / 音频上传
- 批量文件上传
- 文件下载(导出报表、下载附件)
- 拖拽上传
- 截图粘贴上传

判别要点:**用户提交的输入或获取的输出涉及"二进制文件"** ,
而不只是文本数据。

## 必查清单(上传)

### 五个维度的边界

详见 `methodologies/boundary-value-analysis.md` 末尾的"多模态输入的边界"小节,五个维度:

#### 1. 大小维度
- 0 字节(空文件)
- 1 字节(最小非空)
- 系统允许的最大值
- 超过最大值 1 字节
- 极端大值(测试 DoS 防护)

#### 2. 格式维度
- 支持的格式各 1 个代表
- 不支持的格式
- **伪装格式**(后缀和实际内容不匹配)—— 这是关键安全测试点

#### 3. 内容维度
- 空白内容(全白图、空 PDF)
- 极简内容
- 极复杂内容
- 含敏感数据(图含人脸、文档含信用卡号)
- **含恶意 payload**(SVG 中的 XSS、PDF 中的 JS、ZIP bomb)

#### 4. 元数据维度
- 无 EXIF 元数据
- 含 GPS 坐标(隐私)
- 文件名特殊字符(如 `../../../etc/passwd.png` 测路径穿越)

#### 5. 损坏 / 异常维度
- 截断文件
- 头部损毁
- 嵌套压缩炸弹

### 上传过程

- **上传进度显示**:进度条是否准确
- **取消上传**:进行中取消,后端是否清理已收到部分
- **上传失败重试**:是否能从断点续传
- **并发上传多个文件**:互不干扰、整体进度显示
- **慢网络下的上传**:超时设置、用户体验

### 服务器侧验证

- **服务器再次校验**:不依赖客户端的"看起来 OK"
- **病毒扫描**(若有):上传后是否扫描
- **存储路径安全**:上传文件不能存到系统敏感目录
- **文件名重写**:服务器是否生成安全文件名(不直接用用户提供的)

## 必查清单(下载)

### 基础

- **正常下载**:文件正确生成、Content-Type 正确、文件名编码正确(中文文件名)
- **越权下载**:能否下载别人的文件、不该公开的文件
- **不存在文件**:返回 404 而非 500
- **大文件下载**:断点续传(若有)、传输中断处理

### 流式 vs 全量

- **流式下载**(边生成边发送):中途出错如何处理
- **全量下载**(全部生成完才发送):大文件等待时间、超时

### 安全

- **路径穿越**:URL 或参数中的 `../` 是否被正确处理
- **文件名注入**:Content-Disposition 中的文件名是否安全
- **直接 URL 访问**:不经认证能否猜到 URL 直接下载

## 关键提示

**伪装格式**是文件上传最常被遗漏的安全测试。代码用文件后缀判断文件类型 + 不校验 magic number,
直接被绕过。每个上传功能都应该测"`.png` 后缀但实际是 .exe"。

**Chrome 扩展类工具下,文件下载通常无法验证文件内容**——只能验证下载 URL 和 HTTP 头。
这种 Cartographer 阶段 2 要识别工具能力,把"下载内容验证"标 manual_upload 或调整测试方式。

**测试用文件准备策略**(详见 cartographer.md 阶段 2.5):
- A. 用户指定路径(已有 fixture)
- B. 运行时手动上传(用户在测试时手动操作)
- C. Agent 生成(临时生成简单文件,有能力边界)

**注意 Agent 生成的能力边界**:能生成简单图片、损坏文件、伪装文件、纯色块,
但不能生成"真实人脸照片"等需要真实数据的内容。

## 常见叠加

通常和这些模式一起出现:
- 表单输入型(文件作为表单字段之一)
- 多租户 / 权限矩阵(谁能上传谁能下载)
- 异步 / 流式输出(上传 / 下载进度)
- CRUD 列表与详情(文件作为资源被管理)

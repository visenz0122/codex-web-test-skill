# Scenario Patterns(场景模式索引)

## 这是什么

方法论(EP/BVA/Decision Table 等)告诉你**"该测多周全"**——它们是评判覆盖度的标尺。
但同样是 EP,登录功能的等价类和对话框完全不同——**不同场景该重点查的点不一样**。

场景模式库就是**"这一类功能特别要测什么"的清单**,和方法论正交,共同决定测试覆盖度。

## 怎么用

### Cartographer 阶段 1 末尾

识别功能匹配哪些场景模式,写进规约的 `## 4. Scenario Patterns` 字段。
每个模式可叠加(一个对话型 LLM agent 通常匹配 4-5 个)。

### Cartographer 阶段 2

对每个匹配的模式,**打开对应文件**读必查清单,**逐项判断**:

- ✓ 适用且已测 → 自检表标 ✓ + 对应 TC
- ⚠ 适用但没测 → 自检表标 ⚠ + 具体理由
- ✗ 不适用(代码层面不可能)→ 自检表标 ✗ + 代码依据

### Inspector

不"独立列清单",而是**审查 Cartographer 的自检表**(完整性、理由具体性、依据自洽性)。

详见 `references/inspector.md` 第 1.5 步。

## 模式总览

第一版 11 个模式(开放扩展):

| 模式 | 适用场景一句话描述 | 文件 |
|------|----------------|----|
| 表单输入型 | "填表 + 提交" 类:登录、注册、提交资料、密码重置等 | `form-input.md` |
| 用户认证 / 会话管理 | 涉及用户身份建立、维持、终止 | `auth-and-session.md` |
| 个人主页 / 资料管理 | 读取展示用户资料 + 编辑保存 | `profile-management.md` |
| CRUD 列表与详情 | 资源管理类:文章 / 订单 / 商品 / 任务 / 评论 等 | `crud-list-detail.md` |
| 多租户 / 权限矩阵 | 用户、角色、资源隔离 | `multi-tenant-permissions.md` |
| 对话型 UI | 聊天框、客服窗口、AI agent 对话、评论回复 | `dialog-ui.md` |
| 异步 / 流式输出 | 发起请求 → 等待 → 渐进出现:SSE、WebSocket、流式 LLM | `async-streaming.md` |
| LLM agent 决策 | 后端调用 LLM 做决策、生成、对话、工具调用 | `llm-agent-decisions.md` |
| 文件上传 / 下载 | 文件输入或文件输出 | `file-upload-download.md` |
| 状态流转 | 系统在不同状态间切换:订单、token 生命周期、审批流 | `state-transitions.md` |
| **前端渲染保真度** | **后端数据正确但前端渲染错误**(Markdown / 时区 / emoji / 编码 / 数字格式化) | `frontend-rendering-fidelity.md` |
| 异常路径(通用) | 任何功能都该问的:网络、5xx、限流、超时 | `exception-paths.md` |

## 识别提示

读规约的 Behaviors,问自己几个问题:

- 有"输入字段 + 提交"动作吗 → 表单输入型
- 涉及登录态 / token / session 吗 → 用户认证 / 会话管理
- 有"读取 + 编辑保存"用户资料吗 → 个人主页 / 资料管理
- 有列表 / 详情 / 增删改查吗 → CRUD 列表与详情
- 涉及多角色 / 资源隔离吗 → 多租户 / 权限矩阵
- 有聊天 / 评论 / 输入框 + 历史消息吗 → 对话型 UI
- 后端涉及流式输出 / SSE / WebSocket / 长轮询吗 → 异步 / 流式输出
- 后端调 LLM 做决策 / 生成 / 对话吗 → LLM agent 决策
- 有文件上传 / 下载吗 → 文件上传 / 下载
- 有明确状态机吗 → 状态流转
- **展示后端数据给用户看吗(任何形式:文本 / 时间 / 数字 / Markdown / 富文本)→ 前端渲染保真度**(几乎总要加)
- **任何非纯静态功能 → 异常路径(通用)**(几乎总要加)

可叠加。一个对话型 LLM agent 通常匹配:对话型 UI + 异步流式 + LLM agent 决策 + 前端渲染保真度 + 异常路径。

## 添加新模式

如果实测中发现新场景类型(如"实时协作编辑"、"区块链交易"、"图像识别"),按以下步骤追加:

1. 起一个简洁名字(2-4 个字)
2. 在 `scenarios/` 目录下新建一个 `.md` 文件
3. 文件结构按现有模式文件:适用范围 / 必查清单 / 关键提示
4. 在本 `index.md` 表格里追加一行
5. 不需要改其他任何文件——Cartographer 和 Inspector 会自动读取本目录最新版本

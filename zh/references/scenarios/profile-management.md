# 个人主页 / 资料管理

## 适用范围

读取展示用户资料 + 编辑保存的功能:

- 个人主页(展示用户信息)
- 编辑资料(姓名、邮箱、电话、简介等)
- 头像上传 / 修改
- 修改密码
- 通知偏好设置
- 隐私设置

判别要点:**功能聚焦于"用户对自己的数据进行读取和修改"**。
通常不涉及他人数据,但可能涉及他人查看自己。

## 必查清单

### 资料读取

- **登录用户看自己的资料**(成功路径)
- **资料字段完整展示**:每个字段都有展示
- **空字段处理**:用户没填某字段时,UI 如何显示(占位符 / 隐藏 / "未填写")

### 资料编辑

- **每个字段单独修改**:只改一个字段,其他字段不动
- **多字段同时修改**:一次提交多个字段
- **字段约束**:邮箱格式、电话格式、用户名唯一性、长度限制
- **保存后回显**:UI 是否反映最新值(防缓存陈旧)

### 头像上传(若有)

- 详见"文件上传 / 下载"场景模式
- 通常涉及大小、格式、尺寸限制

### 隐私字段

- **修改密码**:要求当前密码、新旧密码不能相同
- **修改邮箱**:是否需要原邮箱验证、新邮箱验证
- **修改手机**:是否需要短信验证

### 越权读写

- **读自己的资料 → ✅ 成功**
- **读别人的资料**:URL 改 ID 直接访问 / API 直调,看是否被拒
- **改别人的资料**:同上,需要服务器端权限校验
- **公开 vs 私密字段**:某些字段是公开的(用户名、头像),某些是私密的(邮箱、电话)

### 并发与一致性

- **同一用户两标签同时修改**:不同字段会冲突吗、同字段会冲突吗
- **未保存离开提示**:修改了字段但没保存,关闭页面 / 跳走是否提示
- **保存成功提示**:UI 反馈是否清晰
- **保存失败处理**:网络断、服务器 500、并发冲突时的体验

## 关键提示

**越权读写**是最容易遗漏的——开发者通常只测"自己看自己",
忘了测"用户 A 改用户 B 的资料"。这个 bug 在生产中很常见,
因为后端经常用前端传的 user_id 而不是 session 里的 user_id。

**未保存离开提示**容易被忽略——用户会因此失去工作进度。

## 常见叠加

通常和这些模式一起出现:
- 表单输入型(每个字段的输入校验)
- 多租户 / 权限矩阵(谁能看谁、谁能改谁)
- 文件上传 / 下载(头像)

## 不适用情况

- 管理员管理其他用户的资料 → 用"多租户 / 权限矩阵"模式
- 公开资料展示给陌生用户(如博客作者主页)→ 偏向"CRUD 详情" + 权限模式
